Intro

Insert file transfer graphic here, reduce text

Say you prepare your presentation on one computer and then try presenting it on another computer. Things that often go wrong:
  • They have a different version of office software running
  • The fonts they have are the same, but somehow configured differently

Sorry!

HackSlider is probably not the right solution for you. You are better off just generating a PDF file. Works better than just about anything else. PDF, however, doesn't work if you:
  • Have one or two slides of animation (please don't overuse this)
  • Are expecting to edit the slides a little after you have transferred it to a different machine
  • Simply do not want to install a full office suite. Those things are bloated!
fix all the line breaks

Upshot

All this specially makes sense (assuming you already know html/css) since modern browsers handle these tasks arguably better than office programs:
  • The document format is well-supported on multiple browsers

    Your slideshow crashes twice per minute on presentation day? Just switch to a different browser

  • Popular browsers are free

    No need to buy another software

  • Browsers often load faster

    Specially if you often have the browser running in background anyway

Getting Started

<script src="jquery.js" type="text/javascript"></script>
<script src="hackslider.js" type="text/javascript"></script>
<link rel="stylesheet" href="hackslider.css" type="text/css"/>
...
<body>
  <div id="slideshow">
      each <div> nested in here 
      automatically becomes a new slide
      ...
  </div>
</body>

Built-in Features

  • Keyboard navigation
  • Automatic slide numbering (can be hidden)
  • Direct links to any slide with #hashtags

Customization

  • Text color, size, font changed using CSS

    Text <span style="red">color</span>

  • Common options are already provied as built-in classes
    • Hiding page number: class="hideSlideNumber"
    • Common slide layouts
  • Just add your own class if things get tedious

    Check the source to see how <style> and <div> can be neatly arranged in pairs for easy editing.

  • Complicated diagrams are just imported as <img>

    Drawing directly in office suites always felt awkward for me anyway. That's not what they are designed for.
